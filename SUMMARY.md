### 3. **JSON スキーマ** - 5つの定義ファイル

#### train_definition.json（89行）
```
列車・系統の完全定義
- systems[]: 複数系統対応
```

**主な特徴:**

```
路線・駅の完全定義
```

**主な特徴:**
- ✅ サンプル駅 5 + 3 + 3 = 11 駅実装
- ✅ 種別ごとの所要時間対応
- ✅ 乗換情報・ホーム案内対応

```javascript

```

**メリット:**

- 言語追加が簡単

### 2. 複数路線参照対応 ✅
系統が複数の route/*.json ファイルを参照：

```javascript
  { lineId: "route/line2.json", name: "高崎線" }

```

**メリット:**
- 直通運転対応
- ファイル管理の柔軟性
- スケーラビリティ

### 3. ユーザー定義可能な設定 ✅

- 言語別周期設定
- インフォメーション表示パターン

**メリット:**
- 小規模～大規模な鉄道網に対応
- カスタマイズ性が高い

## 📝 ローカルテスト方法

### セットアップ

```bash
# ファイルをダウンロード
# (GitHub または outputs フォルダから)

# HTTP サーバー起動
cd /path/to/lcd
python3 -m http.server 8000

# ブラウザで開く
# http://localhost:8000/control.html
```

### 操作フロー

1. **control.html** で：
   - 系統を選択 (東海道線・中央線など)
   - 種別を選択 (普通・快速など)
   - 駅を選択
   - 運行状態を切り替え

2. **「LCD ビューアを開く」** ボタン
   - 新規ウィンドウで viewer.html が開く
   - postMessage でデータ送信開始
   - 画面に情報が表示される

3. **control.html で変更**
   - 駅を変更 → viewer で更新
   - 状態を変更 → viewer でスライドショー切り替え
   - 言語を追加 → 次のサイクルから反映
